- keep it minimal. no upgrades, no classes...polish and quality over quantity

Bullet Patterns
- bullet shape / color should have meaning, e.g. color is speed and shape is targeting logic
    - homing
    - straight line
    - bullets that spawn other bullets
- Modular system
    - spawn position
    - spray pattern
    - fire rate
    - initial rotation
    - bullet type
    - Example:
        - firing two offset circles would be 2 "patterns"; one circle firing at rate N 
        and another also at rate N with offset XX degrees and time offset T/2
- can have a pre-coded pool of bullet patterns, grouped by difficulty level
    - boss then randomly picks from the appropriate difficulty pool
- if needed, can optimize with an instanced/batch 2d sprite shader
- bullet visuals
    - shape
    - size
    - color
    - speed
        - speed over time
    - direction
    - targeting / movement logic
        - bullets that spawn other bullets
        - fixed straight line
        - arc / spiral
        - target player
- change over time. maybe each bullet sequence can have
    - delta_theta
    - delta_cd...
- crazy idea: some bullet trajectories can actually be in 3D space, and project back to 2D for visuals and updating 
  the b2 collider
    - maybe a theme can be 3D-->2D???
